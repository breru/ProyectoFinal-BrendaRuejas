Análisis Preliminar - Proyecto QA - TikTok Lite


Introducción:

El presente documento describe el análisis y alcance del proceso de testing que fue realizado en la aplicación TikTok Lite. Se eligió esta aplicación por contener una gran base de usuarios, y siendo esta una versión optimizada y enfocada a dispositivos con menor capacidad y conexiones limitadas. Esto permite ejecutar casos de prueba funcionales, ya sean estas por cada uno de sus componentes, como también la integración de los mismos. Además permite verificar pruebas del sistema al replicar las condiciones de uso por parte de los usuarios, evaluando los criterios de aceptación en base a los requerimientos prestablecidos por el cliente para el comportamiento de funcionalidades claves como la visualización de contenido, la interacción con videos y navegación dentro de la app.

El objetivo de este proyecto es aplicar metodologías y prácticas de QA a fin de evaluar el correcto funcionamiento de sus principales funcionalidades a partir de las cuales puedan identificarse posibles defectos que sean documentados para analizar los resultados obtenidos. esto incluye planificación de pruebas, diseño de cada uno de los casos de prueba junto con su ejecución y correspondiente reporte de defectos que serán presentados en un informe final. Este objetivo está pensado para verificar las funcionalidades de la app desde el punto de vista del usuario final.



Público a quien está dirigido:

El presente informe estará dirigido a los equipos de desarrollo de software y de QA, Product Owners o responsables del proyecto, como también a las partes interesadas en evaluar la calidad funcional del sistema.


Reglas de negocio consideradas para este proyecto:

1) Los usuarios deben poder registrarse con nombre de usuario válido, formato definido por el sistema.  
2) Cada usuario registrado debe poder iniciar sesión en la app.
3) Cada usuario debe poder visualizar el contenido que el sistema permite crear, como videos que puedan encontrarse dentro del feed principal.
4) Cada usuario debe poder interactuar con otros usuarios y con cada contenido, ya sea mediante la acción de dar "Like", compartir, seguir a otros usuarios y comentar las publicaciones.
5) La sesión debe mantenerse activa mientras el usuario navega por la app, permitiéndole acceder a diferentes secciones (feed, perfil del usuario, búsqueda de contenido)


Requerimientos funcionales:

1) El usuario debe poder registrarse correctamente e iniciar sesión con sus credenciales, como también acceder y visualizar su propio perfil.
2) El sistema debe permitir la visualización del contenido creado por otros usuarios.
3) El usuario debe poder interactuar con otros usuarios y contenidos (dar Like, Seguir) incrementando y expandiendo la red social.
4) El usuario debe poder navegar en las diferentes secciones que ofrece la app.
5) El sistema debe permitir cerrar sesión, editar el perfil de usuario y recuperar su cuenta en caso de olvidar su contraseña.


Requerimientos no funcionales:

1) La app debe cargar el contenido en tiempos razonables.
2) La interfaz debe ser clara e intuitiva.
3) La app debe funcionar correctamente en dispositivos móviles de gama baja o media.
4) La app debe mantenerse estable durante su uso
5) La app debe manejar correctamente cambios de estado
6) La app debe responder y adaptarse correctamente a las configuraciones personalizadas por el usuario



Alcance y limitaciones del proyecto:

En la definición del alcance específico, el testing se centrará en pruebas funcionales manuales desde la perspectiva del usuario. Por lo tanto, no formarán parte del testeo pruebas de seguridad, de algoritmo en la recomendación de contenido, de rendimiento a gran escala, de análisis de código, ni de portabilidad hacia otros dispositivos.


Observaciones y aclaraciones:

Las pruebas se realizarán sobre la versión pública de la app que se encuentra disponible al usuario. Cabe destacar que alguna funcionalidades dependerán de la conectividad y/o configuraciones del dispositivo móvil. En cualquier caso, se documentarán los defectos encontrados con evidencia visual cada vez que sea posible.



Épica: testeo de funcionalidades principales para la aplicación TikTok Lite

HU1: Como usuario nuevo quiero poder registrarme en la app para crear mi propia cuenta y acceder a sus funcionalidades
	C1: el sistema debe permitir ingresar un nombre de usuario válido
	C2: el nombre de usuario debe tener más de un carácter
	C3: el nombre de usuario no debe contener mayúsculas ni caracteres especiales que no sean puntos y guiones bajos
	C4: el susuario ebe poder ingresar una contraseña válida, entre 8 y 20 caracteres, con letras, números y caracteres especiales
	C5: el sistema debe poder mostrar una leyenda con los requisitos de validación para el nombre de usuario y la contraseña
	C6: el sistema debe poder rechazar los datos ingresados cuando no cumplen con los requisitos establecidos

HU2: como usuario quiero iniciar sesión para acceder a mi cuenta
	C1: el sistema debe permitir ingresar las credenciales correctas
	C2: el sistema debe permitir al usuario ingresar al feed principal 
	C3: el sistema debe mostrar un mensaje de error cuando las credenciales son incorrectas
	C4: el sistema debe rechazar el inicio de sesión con credenciales vacías  

HU3: como usuario quiero ver videos en el feed para consumir contenido de manera rápida
	C1: el usuario debe poder abrir la app
	C2: los videos deben poder cargarse sin demoras
	C3: el usuario debe poder desplazarse por el feed para ver el contenido
	C4: los videos deben poder visualizarse correctamente sin errores visuales

HU4: Como usuario quiero interactuar con el contenido de otros usuarios mediante la acción "Like"
	C1: el usuario debe poder acceder al botón "Like", visualizado en cada uno de los videos del feed
	C2: el botón "Like" debe responder al presionar el ícono en pantalla
	C3: el ícono debe cambiar de estado indicando la concreción de la acción "Like"

HU5: Como usuario quiero poder seguir a otros usuarios para ver sus contenidos e incrementar mi red
	C1: debe existir un botón para la acción "Seguir" para cada usuario de la plataforma
	C2: el sistema debe registrar el seguimiento una vez realizada la acción
	C3: el botón "Seguir" debe cambiar de estado para cada usuario a "Siguiendo"

HU6: como usuario quiero navegar por las distintas secciones y pantallas de la aplicación y acceder a sus funcionalidades
	C1: el usuario debe poder acceder a su perfil
	C2: el usuario debe poder volver al feed principal
	C3: el usuario debe poder acceder a la bandeja de entrada
	C4: el usuario debe poder ingresar al listado de contactos sugeridos para seguir
	C5: el usuario debe poder acceder a los contenidos likeados 
	C6: el usuario debe poder acceder a los contenidos privados
	C7: el usuario debe poder acceder a los contenidos guardados
	C8: la navegación entre pantallas debe ocurrir sin errores ni demoras

HU7: como usuario quiero que la aplicación mantenga su estado al pasar a otra aplicación y regresar nuevamente
	C1: la aplicación debe conservar su estado si pasa a 2do plano
	C2: la aplicación debe mostrar la última vista, previa al paso a 2do plano
	C3: la aplicación no debe reiniciarse
	C4: la aplicación no debe cerrarse

HU8: como usuario quiero visualizar la configuración personalizada y aplicada a la aplicación
	C1: si el dispositivo está configurado en modo oscuro, la aplicación debe respetar esa configuración
	C2: si la configuración cambia al modo claro, la aplicación debe modificar la misma
	C3: no debe haber inconsistencias visuales
